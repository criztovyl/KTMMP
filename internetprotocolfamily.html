Die Internetprotokollfamilie

Das Internet besteht aus ganz unterschiedlichen Protokollen, die dann unter diesem Namen zusammengefasst sind. Hier ein paar Beispiele.
Aufgrund der enormen Wichtigkeit von TCP und IP, wird mit TCP/IP oft die gesamte Internetprotokollfamile bezeichnet. Dazu gehören aber nicht nur TCP und IP.

  • FTP, das File Transfer Protocol
  • IMAP, das Internet Message Access Protocol um auf eMail-Postfächer zuzugreifen
  • SMTP, das Simple Message Transfer Protocol um eMails zu senden
  • SSH, das Secures Shell Protokoll, für eine gesicherte Verbindung zu entfernte Kommandozeile
  • UDP, das User Datagram Protocol, die verbindungslose Variante von TCP
  • TLS, das Transmission Layer Security (war früher SSL) für verschlüsseltes TCP

HTTP

HTTP, das "Hypter Text Transfer Protocol" ist das Protokoll, das den "Hyper Text" (Hypertext) über TCP transportiert. Der Hypertext zeichnet sich dadurch aus, dass er mit anderen Hypertexten verbunden ist. Der englische Begriff dafür ist "to reference" oder "to link" und da er im Hypertext zu finden ist, heißt er "Hyper Reference" (href) und "Hyper Link" (Hyperlink), der mittlerweile einfach nur unter dem Namen "Link" firmiert.

IP

IP ist für die Adressierung der Netzteilnehmer, im Fachjargon auch "Host", zuständig. IP wird zum Beispiel von TCP genutzt. Möglicherweise hast du sogar schon einmal eine IP Adresse gesehen, ein typisches Beispiel für eine IP Adresse ist 192.168.X.1, was meistens die Adresse deines Routers ist. Derzeit gibt es zwei Versionen von IP: Version 4 und Version 6. Die Beispiel-IP ist eine IPv4 Adresse, eine IPv6 Adresse sieht z.B. so aus: 

fe80::325:23ff:fef8:fc07

TLS

TLS wird dazu verwendet, um Verbindungen zu verschlüsseln. Es findet aber nur ein bestimmter Teil der Kommunikation (zwischen Computern) im Internet verschlüsselt statt. Verschlüsselt sind z.B. Bankseiten oder andere, wichtige Seiten. So ist es zum Beispiel möglich, dein Passwort, dass du auf einer Webseite eingibst, die kein HTTPS unterstützt, abgefischt werden kann. Ein großer Problem der Seitenbetreiber sind aber meist die hohen Zusatzkosten, um HTTPS anzubieten.
