HTTP

HTTP, das "Hyper Text Transfer Protocol", wird genutzt, um Hypertext zwischen Computern zu übertragen. HTTP ist ein Protokoll, dass vollständig auf reinen Text ("plain text") setzt. Am bekanntesten aus diesem Protokoll ist ein Fehler: HTTP/1.1 404 Not Found. Die typische Strukur einer HTTP-Anfrage sieht so aus: GET / HTTP/1.1.

HTTP-Request
HTTP Anfragen

  • GET: Fordert eine Ressource an. Es können Wertepaare übergeben werden, in dem sie als Argument=Wert, mehrere mit & getrennt, mit vorrangehendem ? an die URI gehängt werden. Bsp.: GET /index.php?seite=start&user=trulla HTTP/1.1
  • HEAD: Fordert die Kopfdaten einer Ressource an
  • POST: Schickt Daten zur weiteren Verarbeitung an den Server. Argumente werden an den Server übergeben, in dem sie wieder als Argument-Wert-Paare übergeben werden, wobei mehrere wieder mit einem & getrennt werden. Diesmal werden sie aber nicht an die URI anghangen sonder werden im Header mit übergeben (siehe auch unten).
  • PUT: Dient dazu, eine Ressource auf die Angegebene URI hochzuladen (meist nicht mehr verwendet)
  • DELETE: löscht eine Ressource (meist nicht mehr verwendet)
  • TRACE: liefert genau die Anfrage zurück, die abgeschickt wurde
  • OPTIONS: eine Liste von Fähigkeiten des Servers
  • CONNECT: für Proxyserver mit SSL(TLS)-Tunneln

HTTP Header

Der HTTP-Header besteht aus mehreren Zeilen, die jeweils ein Wertepaar enthalten. Hier wird bei GET zum Beispiel der Ziel-Host angegeben. Bei POST werden hier auch noch die Argumente übergeben, wobei deren Typ und deren Länge angegeben werden muss.
Bsp:

GET /index.php HTTP/1.1
Host: example.com
Content-Type: application/x-www-form-urlencoded
Content-Length: 43
page=start&user=trulla&action=play
