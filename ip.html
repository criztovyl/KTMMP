IP
Internet Protocol

Das Internet Protokoll ist für die Addressierung der einzelnen Hosts verantwortlich und gewährleistet das Routing. Derzeit gibt es zwei Versionen von IP, IPv4 und IPv6. Der größte Unterschied zwischen beiden ist, dass sie eine andere Anzahl an möglichen Adressen haben: IPv4 hat nur 232 (4,3*109) Adressen, IPv6 schon 2128 (3,4*1038).

IPv6

Darstellung

IPv6 wird hexadezimal in 8 4er Gruppen, getrennt durch :, dargestellt. Hierbei könne vorangehende Nullen weggelassen werden und Gruppen die "0" sind, ein Mal zu "::" zusammengefasst werden. Bsp.: 2a03:4000:1::2e26:eee7:1

Header

from netzmafia.de

Der IPv6 Header hat nur sieben statt bisher 13 Felder.
  • Version: Die Version, bei IPv6 entsprechend 6, bei IPv4 4. So können v6 und v4 nebeneinander verwendet werden.
  • Priority: Die Priorität des Pakets, bei 0 nicht definiert, bei 1-7 nicht Echtzeit und 8-15 Echtzeit. Je höher die Priorität, desto mehr wird das Paket bevorzugt behandelt.
  • Flow Label: Alle Pakete mit dem gleichen Flow-Label nehmen den gleichen Weg.
  • Paylaoad Length: Nutzlastlänge, die dem Basisheader folgt.
  • Next Header: Der Erweiterungsheader, der dem Basisheader folgt.
  • Hop Limit: Dieses Feld wird nach jedem "Hop" um eins vermindert. Wenn es 0 ist, wird das Paket verworfen.
Bei IPv6 gibt es bisher sechs Erweiterungsheader, die nicht von jedem Router behandelt werden müssen, außer dem Hop-by-Hop Options Header.
